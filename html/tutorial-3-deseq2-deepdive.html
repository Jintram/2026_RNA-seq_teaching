Normalization: size factors

A simple approach

If sample 1 in total has 5000 reads, and sample 2 in total has 10000 reads (a higher “sequencing depth”), and we consider a gene X, we expect that gene to have more reads in sample 2. In fact, a good guess would be that gene X has 2 times as many reads in sample 2. This is of course not due to biology, but due to the fact that we sequenced sample 2 more.

So, to more honestly compare samples, we need to normalize for the amount of reads in each samples.

When we have count table where rows correspond to genes, and columns to samples, we could simply divide each column by its total, and work with those values. Or, one step better, do that, and multiply with the mean of total sample counts, such that the values we end up with are in the same range as those with which we started.

DESeq2’s approach

DESeq2 takes a little more sophisticated appraoch. They make the argument that we’re intersted in genes that show changes due to regulation.

“The total number of reads, \(\sum_i k_{i,j}\), may seem to be a good measure of sequencing depth and hence a reasonable choice for \(s_j\). Experience with real data, however, shows this not always to be the case, because a few highly and differentially expressed genes may have strong influence on the total read count, causing the ratio of total read counts not to be a good estimate for the ratio of expected counts.” – (Anders & Huber, 2010))

When there are a few highly-abundant genes are upregulated, the concentrations of the other genes in the cells will go down, as the size of the total transcriptome remains similar. (And conversely, if highly-abundant genes go down, other genes’ concentrations will go up.)

DESeq2 thus tries to normalize both for sequencing depth, and this second order effect. It assumes that most gene don’t change, and calculates the median change in counts over all genes as a ‘size factor’ that reflects sequencing depth and the second order effect. Changes in gene expression that remain left after normalization with the size factor then should reflect change that we’re intersted in.

Technically, DESeq2 calculates a pseudo-reference sample, with arithmetic averages over the samples for each gene, then normalizes each gene by the reference sample, and then calculates the median normalized expression per sample. This somewhat complex scheme is summarized below:

  # Calculate geometric means of gene values

  mutate(geom_mean = exp(mean(log(counts)))) |>

Calculate dispersions

What’s dispersion and why is it important

As mentioned, in the end, our aim is to know how changes in gene expression depend on the different conditions of our experiment.

Due to technical and biological factors, gene expression counts will vary, even within the same condition. Dispersion (variability) refers to the extent by which observations of a quantity, in this case counts, vary.

In the example with gene A below, we can say with high likelihood that the gene is differentially expressed, whilst in the example with gene B, we can be less sure of that.

As you can see from the example above, dispersion (variability) plays a big role in determining whether we can trust the fold change estimate. Indeed, dispersion estimates play a central role in DESeq2’s ability to calculate stastistics that inform us about the extent and significance of differential gene expression.

Estimating dispersion plays a large role in DESeq2

A load of code in DESeq2 is dedicated to estimate the dispersion (variability) in each gene’s expression that is not due to differences in experimental conditions. In broad strokes, this estimated dispersion for each gene is then used to:

  • Inform statistical testing. At the end of the DESeq2 procedure, a p-value is calculated to determine statistical significance of the differential expression. In the calculation of this p-value, an estimate of the dispersion is used.
  • Improve L2FC estimates. DESeq2 will estimate the fold-change, which is based on the data, but variability estimates are used to refine the estimate.
  • Improve PCA accuracy. Optionally, you can create a modified gene count matrix in which dispersion information is to normalize the dispersion. When you use a PCA for data visualization, basing it on the variance normalized (“variance-stabilized”) count matrix, will result in a more accurate representation of the data by the PCA.

Obtaining good dispersion estimates

Specifically, DESeq2 assumes gene count values show statistics that are consistent with a negative binominal distribution. For such a distribution, the variance and mean are related by the following relationship:

\[ \text{variance} = \mu + \alpha \cdot \mu^2 \]

With \(\alpha\) a free parameter that determines how the relationship looks precisely. Let’s see how consistent the data is with this assumption:

# Fit alpha

# The function describing variance = mu + alpha * mu^2
fn_NBvariance <- function(mu, alpha) {
    return(mu + alpha * mu^2)
}

# Let's first fit the dispersion coefficient
# Fit alpha in: variance = mu + alpha * mu^2
# Use nls on log-transformed values for a better fit across the wide range
df_fit <- df_dispersion_real %>% filter(mean > 0, variance > 0)
fit <- nls(log(variance) ~ log(mean + alpha * mean^2),
           data = df_fit,
           start = list(alpha = 0.01),
           control = nls.control(maxiter = 200))
MY_ALPHA_real <- coef(fit)[["alpha"]]
cat("Fitted alpha:", MY_ALPHA_real, "\n")

This seems to work rather well for our data. Using this information, we can now estimate the variability of any gene, given its mean \(\mu\):

\[ \text{estimated variance}_\text{gene} = \mu_\text{gene} + \alpha_\text{global fit} \cdot \mu_\text{gene}^2 \]

This is great! If we now have a gene whos average expression is different between two conditions, we can use this information to estimate how likely it is that that difference was due to chance.

However, we want to calculate the likelihood that the mean of multiple FLT count observations are higher (or lower) than the GC mean (reference condition). To do this, we can use a Wald or t-test.

These are very related, both have the same test statistic:

\[ \frac{\mu_{GC} - \text{mean}_\text{FLT}}{\sqrt{\text{variance}}/\sqrt{N}} \]

Here, \(\mu\) indicates our H0 population mean, which in practice will be the mean, \(\text{mean}_\text{GC}\). \(\text{variance}\) is the variance in the counts. Note that \(\sqrt{\text{variance}}/\sqrt{N}\) equals the standard error \(\text{SE}\), which will be relevant later.

In the t-test, this test statistic is assumed to follow the t-distribution. In the Wald test, this test statistic is assumed to follow a normal distribution. The location of the test statistic on the curve indicates the probability of H0 being true, and thus the statistical significance of the observed difference between conditions.

Some technical notes

The Wald test vs. the t-test

As mentioned, the distribution used for the t-test is the t-distribution, and distribution used during the Wald test is the normal distribution.

The central limit theorem states that if we have enough samples, the mean of will be normally distributed, even if the data follows another distribution (such as the negative binominal distribution, in our case).

This is why we can use the Wald test.

However, the central limit theorem breaks down for small sample sizes. The t-distribution accounts for small sample sizes (gets broader), and is usually more suitable for experiments with small sample sizes. For large sample sizes (\(n \to \infty\)), the t-distribution is equal to the normal distribution. Thus, the t-test is a refinement of the Wald test.

For small sample sizes, in principle, the Wald test is not so suitable. However, since DESeq2 doesn’t base the estimated variability and expression difference (reflected by the fold change) on information only from a single gene, it has more reliable variance estimates, such that we can use the Wald test, and get more statistical power.

Technical: the negative binominal distribution is discrete

DESeq2 works with negative binomial distributions, which describe the statistics of discrete numbers.

After normalization of the counts by the size factors, the counts are not integers any more.

Therefor, under the hood, DESeq2 keeps working with the discrete counts.

For reference, the model is written out below, but the terms used are explained in the next section.

\[ K_{ij} \sim \text{NB}(\mu_{ij}, \alpha_i) \] \[ \log(\mu_{ij}) = \log(s_j) + \mathbf{x}_j \boldsymbol{\beta}_i \]

Which can be written as: \[ K_{ij} \sim \text{NB}( s_j e^{\mathbf{x}_j \boldsymbol{\beta}_i} , \alpha_i) \]

Here, \(K_{ij}\) are the raw counts for gene \(i\) in sample \(j\) that are fitted (“\(y\)” from “\(y = \beta_0 + \beta_1 * x\)”), with \(s_j\) the size factor (and \(x_j\) the condition, and \(\beta_i\) the coefficients that we want to estimate).

\[ e^{\mathbf{x}_j \boldsymbol{\beta}_i} \]

represents our quantity of interest, the size-corrected mean counts that we can compare between samples. (The factor \(s_j\) scales that term back to the raw counts.)

Thus, the size factors are included in the fit.

Technically, this looks like:

MASS::glm.nb(counts ~ condition + offset(log(size_factor)), data = df)

What DESeq2 actually does

As will be explained more extensively below, DESeq2 doesn’t simply compare the means of two conditions.

The L2FC equals \(\beta_1\) from a linear model

It instead fits a linear model to the log-transformed counts. In this section we’ll focus on the p-value calculation in DESeq2, so let’s forget about more complicated linear models
and look at a simple example (see the next section that explains DESeq2 uses a GLM and uses dispersion information during fitting):

\[ \text{[mRNA]}_i = e ^ {\beta_{0,i} + \beta_{1,i} x} \]

With \(\text{[mRNA]}\) the concentration of mRNA of gene \(i\) in condition \(x\) (with different conditions represented by numbers, \(x = \{0, 1\}\)), \(\beta_{0,i}\) base level expression, and \(\beta_{1,i}\) the effect of the condition on the expression of gene \(i\). The \(\beta\) constants are the ones that are fitted by DESeq2. The e power is used since DESeq2 works in log-space, which is more suitable for gene expression data.

When calculating the log fold change between two conditions, this amounts to:

\[ log(\text{FC}) = log(\frac{e ^ {\beta_{0,i} + \beta_{1,i} 1} }{e ^ {\beta_{0,i} + \beta_{1,i} 0} }) = log ( e ^ {- \beta_{1,i}} ) = \beta_{1,i} \]

This shows that \(\beta_1\) actually equals the log-fold change. A simple division can convert this to log2 scale, \(\text{L2FC} = \beta_1 / \log(2)\).

The Wald test is based on \(\beta/\text{SE}(\beta_{1,i})\)

The test statistic in DESeq2 therefor is:

\[ \text{test statistic} = \frac{\beta_{1,i}}{\text{SE}(\beta_{1,i})} \]

Where the \(SE\) can be calculated as it is related to the gene-specific variance. The math they use is more advanced, but we saw something similar earlier with \(\text{SE} = \sqrt{\text{variance}}/\sqrt{N}\).

The variances are estimated using Bayesian logics

In the text above, we used a gene specific prediction for the variance:

\[ \text{estimated variance}_\text{gene} = \mu_\text{gene} + \alpha_\text{global fit} \cdot \mu_\text{gene}^2 \]

DESeq2 has a more sophisticated approach that consists of three steps:

1. Gene-wise estimates.

Based on each gene’s observed mean and variance, the \(\alpha_i\) parameter is estimated for each gene. This is done using a generalized linear model (with a design given by the user), which includes the negative binomial probability distribution. Since this also includes \(\beta\) fits according to the model, \(\alpha_i\) can be fitted using information from the multiple conditions, without inter-condition expression differences inflating the variance.

In the text above, we estimated \(\alpha_i\) directly using the within-condition variances and means. DESeq2’s approach with the generalized linear model (GLM) is more precise.

2. Fit a trend.

The second step is one we also did.

DESeq2 fits a curve through the gene-wise dispersion estimates as a function of the mean, capturing the overall mean–dispersion relationship.

3. Shrink toward the trend.

Using a Bayesian approach, which involves updating an initial estimate of a parameter by combining it with additional information (in the form of another probability distribution), DESeq2 now updates it’s initial estimate of the gene-wise dispersion.

Generally, Bayes theorem says: \[ P(\text{A} | \text{B}) = \frac{P(\text{B} | \text{A}) \cdot P(\text{A})}{P(\text{B})} \]

The DESeq2 paper gets very mathematical here, so the following part leans a bit too much on Claude, but is consistent with the paper (disclaimer!) (see equation 9 in the DESeq2 paper).

Applied to the gene expression model, this means \[ P(\alpha_i \mid \text{data}) \propto \underbrace{P(\text{data} \mid \alpha_i)}_{\text{likelihood}} \times \underbrace{P(\alpha_i)}_{\text{prior}} \]

Where \(P(\text{data} \mid \alpha_i)\) is the belief we have about \(\alpha\) from the fit of our data (the GLM), and \(P(\alpha_i)\) is a normal distribution based on the fitted \(\text{variance}(\mu)\) trend.

Final estimate. Finally, this results in a “shrinking” of the dispersion estimate (\(\alpha_i\)) towards the trend. Especially for low certainty estimates (e.g. genes with few counts), the estimate is pulled more strongly towards the trend, while for high certainty estimates (e.g. genes with many counts), the estimate is pulled less strongly towards the trend. After this Bayesian correction, we have the final dispersion estimates.

Relationship to the equation in the paper

To estimate the peak of the final probability, and thus the final dispersion estimate, the mode is used (“MAP”), hence (equation 9 in the DESeq2 paper): \[ a_i^{MAP} = arg max_\alpha (\ell _{CR} (\alpha ; \mu_i, K_i) + \lambda_i(\alpha)) \] where \(\ell _{CR} (\alpha ; \mu_i, K_i)\) corresponds to \(P(\text{data} \mid \alpha_i)\), where the placement of \(\alpha\) is flipped since this is likelihood notation, which still relates to the same probability, but emphasizes that we’re scanning over possible values of \(\alpha\). (The likelihood conveys “given fixed data, which \(\alpha\) is most likely?”) The second function, \(\lambda_i(\alpha)\), corresponds to \(P(\alpha_i)\), the prior. This is a simple Gaussian function, Equation (5) in the DESeq2 paper spells this out. The width of that Gaussian is also based on the width observed in the \(\text{variance}\) - \(\mu\) relationship.

However, to be more precise and versatile (allowing for multiple conditions, e.g.), DESeq2 fits the log2 fold changes using a “GLM” (generalized linear model), which is an extension of a linear model that allows for non-normal distributions (DESeq2 assumes a negative binominal model for gene counts).

For example, if we have the condition \(x\) corresponding to either ground control (\(x=0\)) or space flight (\(x=1\)), we can fit gene expression \(q_i\) for any gene \(i\) with

\[ q_i = \beta_{0,i} + \beta_{1,i} x \]

The idea here is to fit \(\beta\) such that it reflects the effect the condition \(x\) has on the final expression of gene \(i\).

We can perform the GLM fitting procedure as follows:

Technical notes

MASS::glm.nb() uses a “link” function to couple the output of the linear model to the prediction, and per default it takes the log function, such that what we’re fitting will be \(log(y) = \beta_0 + \beta_1 * x_{\text{condition}}\) (f(z) = log(z) being the link function here). In addition, MASS::glm.nb() (or lm() in general) will dummy-code the condition column, such that GC will be coded as 0, and FLT will be coded as 1 (alphabetical order). Both these default behaviors also apply in DESeq2.

Since \[ \text{ln}(y_\text{FLC} / y_\text{GC}) = ln(e^{(\beta_0 + \beta_1 * x_\text{FLC})} / e^{(\beta_0 + \beta_1 * x_\text{GC})}) = \beta_1 \] with \(y\) begin gene expression, the coefficient \(\beta_1\) directly reflects the log fold change between conditions (and \(\beta_1 / log(2)\) the L2FC). Likewise, the intercept \(\beta_0\) reflects the baseline expression (log scale) in the reference condition (GC).

Shrunken L2FC estimates

“A common difficulty in the analysis of [high-throughput sequencing] data is the strong variance of LFC estimates for genes with low read count. (..) weakly expressed genes seem to show much stronger differences between the [conditions] than strongly expressed genes” – (Love, Huber & Anders, 2014)

To counter the effect of blown-up L2FC values in lowly-expressed genes, DESeq2 offers a correction on top of the L2FC estimated by the GLM. The function lfcShrink() applies a “Bayesian” statistics approach, meaning that they re-calculate the estimated L2FC, taking into account the assumption (and corresponding probability distribution) that most genes will have an L2FC of zero, and substantial variability around that zero.

More precisely, a prior is used that is centered at zero (assumption: most genes aren’t DE). Its width is learned from the data, such that genes with weak evidence (low counts → wide likelihood) get pulled more strongly toward zero whilst genes with strong evidence (high counts → narrow likelihood) are barely affected (this is called an empirical Bayes approach, as the prior is based on the data itself).

Calculating p-values (statistical testing)

To achieve this, simply put, DESeq2 will estimate the dispersion (variability) for each gene that is attributed to “randomness”, such that it can estimate whether observed changes lie outside the expected random variability, and are thus likely to be related to the different experimental conditions.

Or, in more technical terms, it will calculate p-values based on dispersion estimates, which reflect the likelihood that H0 (no difference between conditions), is true. If that probability is low (typically \(p<0.05\) or \(<0.01\)), we can reject H0, and say that there is likely a difference between conditions.

The method by which this is done is quite a bit more complicated, but that’s the gist of it.
